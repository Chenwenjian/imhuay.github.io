利用 Hexo 快速搭建自己的博客

1. 安装依赖

  • 安装 Nodejs
  • 安装 Git
  • 安装 Hexo
    > npm install -g hexo-cli

2. 初始化 Hexo

3. 生成并测试本地服务

默认端口号为 4000，修改以防被占用

4. 安装 Next 主题（可选）

5. 发布到 GitHub

  • 创建仓库 username.github.io
  • 安装 Git 依赖
    npm install hexo-deployer-git --save

  • 修改站点配置文件 hexo/_config.yml

    注：这里 repo 的写法千奇百怪，为了避免各种无法访问的错误，直接加上用户名和密码
    该文件不会被推送到 github 上

    知乎/ https://www.zhihu.com/question/38219432/answer/328799436

  • 生成并上传

    或
    > hexo d -g

我为什么要开始写博客

开始的目的很单纯，就是为了能在简历上加上一条——我的博客: …

其实我一直有做笔记的习惯，早前也用 Jekyll 搭建过一个博客，还因为各种坑浪费了不少时间。

也可能是我太喜欢做笔记了，而用博客记录的效率又太低，所以很快就不了了之了…

在此之前，我的笔记主力是 OneNote, 当然它现在也是我最常用的笔记工具。OneNote 的强大之处这里不多赘述，现在我的 OneNote 已经超过 1G。如果全部使用博客来记录，不知道达到这个数量需要多少时间。

不过 OneNote 的缺点随着笔记的增多也渐渐显现，那就是笔记太细之后，很难快速找到重点。

当然，如果你没有强迫症的话，完全可以依靠 OneNote 强大的搜索功能。只要你记得笔记的关键字，它甚至可以对图片中的文字建立索引

所以建立博客的目的有多了一个，那就是——把值得常记的内容放到这上面，可以随时浏览

这里又体现出 OneNote 的一个缺点，那就是移动端的体验不佳
