局部敏感哈希 (Locality-Sensitive Hashing, LSH)

问题背景：海量数据快速查找

常用技术：最近邻查找（Nearest Neighbor），如 kd-tree；近似最近邻查找（Approximate Nearest Neighbor, ANN），如各种kd-tree 的衍生。LSH 属于 ANN 中的一种技术

与传统 Hash 不同，LSH 的基本思想是：两条原本相似/相邻的数据在经过相同的 Hash 后，依然相似/相邻的概率尽可能大，这期间可能伴随着维度的降低。也就是说，我们希望原先相邻的两个数据能够被 Hash 到相同的区域（Hash 桶）。这样在查找时，直接定位到某个 Hash 桶内，进而找到所有可能相似的信息。

LSH 使用的 Hash 函数需要满足以下两个条件：

1. 如果 d(x,y) ≤ d1，则 h(x) = h(y) 的概率**至少**为 p1
2. 如果 d(x,y) ≥ d2，则 h(x) = h(y) 的概率**至多**为 p2
