fork 一个项目后该怎么做

发表于 2018-04-10

初学 git - fork 一个项目后该怎么做？

如何保持仓库副本、本地副本 与 源仓库同步更新？

测试项目：

  • 源项目： https://github.com/imhuay-m/learn-git
  • 副本仓库： https://github.com/imhuay/learn-git

开始

  1. fork 源仓库 -> 在自己的 GitHub 上生成一个副本仓库
  2. clone 副本仓库到本地

如何保持仓库副本、本地副本 与 源仓库同步更新？

首先看下远程仓库的信息

如果使用 GitHub 客户端 clone 副本仓库，那么会自动添加上流(upstream)，即源仓库地址

如果是使用 git clone 的，则不会

假设使用的是 git clone，那么第一步就是要手动添加 upstream

这时再查看远程仓库

删除 upstream：git remote rm upstream

此时，就可以合并源仓库了

此时，知识更新了本地仓库，而远程的副本仓库还没有同步，还要 push 一下

测试

副本仓库没有更新，源仓库更新了

同步成功

本地仓库添加了新的文件，源仓库没有更新

更新成功

副本仓库的修改不会影响源仓库

本地仓库和源仓库修改了不同的文件

更新成功

git merge upstream/master 不能写成 git merge master/upstream

本地仓库和源仓库修改了同一文件，有冲突

步骤是一样的，略

这种冲突是无法自动解决的，先记录下发生冲突的提示

看一下发生冲突的文件README.md

<<<<<<< HEAD 表示本地的修改（当前更改）

>>>>>>> upstream/master 表示来自源仓库的修改（传入的更改）

这里保留来自源仓库的修改，更改后需要先 git add 和 git commit，然后再git merge就能成功了

一些编辑器/IDE可以帮助你快速处理这些修改，比如 VSCode

本地仓库和源仓库修改了同一文件，但是没有冲突

测试了一下， git 识别的并不十分准确，比如只是新增了一个空行，应该是不冲突的，但是依然发生了冲突。

这个还是情况处理吧。

注意：一旦你处理了这个冲突，无论你做了什么修改，git 将不再处理该冲突，即使与源仓库不一致。
