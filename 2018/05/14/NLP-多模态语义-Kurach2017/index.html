（文献阅读）多模态语义之 Image-to-Text Transfer

发表于 2018-05-14

多模态语义相关文献阅读

文献名称: Better Text Understanding Through Image-To-Text Transfer

摘要（Abstract）

泛型文本嵌入（Embedding）已经成功地用于各种任务中。然而，它们往往是通过从纯文本语料库中捕获共现结构来学习的，从而限制了它们的泛化能力。本文探索了将视觉信息纳入文本表示的模型。基于 comprehensive ablation studies（控制变量法），我们提出了一个概念简单，但性能良好的结构。在一套既定的基准上，它优于以往的多模态方法。我们还改进了图像相关文本数据集（image-related text datasets）的最新成果，使用数量级较少的数据。

前言（Introduction）

如何有效表示文本数据一直是 NLP 的研究重点。许多模型能够通过有监督的端到端结构来直接学习。然而，这往往需要大量的标记数据，这意味着高昂的成本，甚至在一些领域无法做到。一个常见的解决办法是重用预先训练过的 embeddings。

这一领域的大量工作集中在从纯文本数据中训练 embeddings。然而，有许多其他类型的关系和共现是很难从纯文本中理解的。这些关系有更自然的表现方式，比如图像。

比如，从图像空间中的相似性度量以及（图像，句子）对中，可以导出句子的相似性度量，如图所示。

这两幅图像在视觉空间中是比较接近的，这可以通过 CNN 来量化。他们的描述传达了相同的概念，但使用了完全不同的词汇。该方法利用图像的视觉属性知识，提高了对文本的理解。

本文研究了如何从文本图像对构建句子嵌入（Sentence Embedding），这些文本图像对在句子相似度量方面是很好的。这个工作拓展自之前的研究 [12] 和 [15]

本文提出了一个概念简单但性能良好的模型——Visually Enhanced Text Embeddings（VETE），该模型利用图像中的视觉信息来提高 Sentence Embedding 的质量。

模型的结构很简单：使用预训练的卷积模型计算 image embedding，sentence embedding 则通过归一化词向量的和得到。然后对两者进行某种组合。

这个模型最大的优点在于使用很少的训练数据就达到了目前的最佳结果。这表明来自图像的信息可以显著提高 text embedding 的质量。

本文工作小结：

  • 提出了一种简单的多模式模型，其在各种文本相似性任务上的表现胜过了之前的图像-文本方法，且使用了更少的数据进行训练
  • 本文对图像到文本的迁移（image-to-text transfer）、不同的模型类型（different model
    types）、文本编码器（text encoders）、损失类型（loss types）和数据集（datasets）进行了全面的研究。
  • 直接学习 sentence embedding 优于词向量组合的方式。

相关工作（Related Work）

多数工作的研究方向是如何将多模态数据（主要是 text-image 对）应用到相关的任务中，比如图像描述（image captioning）、图像检索（image retrieval）等，这些任务需要模型同时了解 text 和 image 相关的知识。

[12]拓展了 skip-gram 算法，并使用 max margin loss 来向词向量中注入图像信息（具体参考原文）。[12]证明了加入视觉信息可以增强词嵌入的表达，具体体现在单词相似性任务上有更好的性能。

[11]也使用了 max margin loss 来 “co-embed” images 和 text。并且得到了一些类似词嵌入的算术性质：”image of a blue car” - “blue” + “red” = “image of a red car”。

最近的一些工作开始从文本相似性的角度探究图像信息对于 phrase embeddings 的提升（从 word 到 phrase）。比如，[15]使用 RNN 来学习基于词嵌入的 phrase embedding，并提出了三个模型。本文使用[15]中的模型A 作为 BaseLine。

本文模型 - VETE Models

模型的目标是在图像和文本表示之间直接传递知识，其主要目的是找到可重用的句子嵌入。

模型的输入是（图像，文本）对，其中文本是一个描述图片的句子。

图像被送入一个预训练的 CNN 模型 $E_{img}$ (encoder of image embedding)。而句子由模型 $E_{txt}$ 进行编码。因为 image embedding 的维度通常会比 sentence 的大很多，所以 image embedding 需要经过一个仿射变换矩阵 $W$ 来修改其输出维度以匹配 sentence embedding。最后将两个 embeddings 分为两组：matching pairs 和 incorrect pairs。分别计算它们的相似度，并以 Pearson correlation（皮尔逊系数）作为损失函数。其中只有绿色的部分会被训练。

$E_{txt}$ 中的 BOW 指的是 normalized sum of vectors，而不是传统的词袋模型；RNN 基于 LSTM 或 GRU；CNN 用的是 [9] 中的结构；实际模型分为 VETE-BOW, VETE-RNN 和 VETE-CNN。

$E_{img}$ 使用的是 InceptionV3，输入为 300x300 的图像，输出一个维度为 2048 的特征向量

本文使用 consine 作为相似度计算函数 $sim(e_i,e_t)$。模型的目标是最大化该相似度如果 image 与 sentence 是匹配的，反之最小化该值。但这不是模型的目标函数，目标函数为最大化皮尔逊系数 $p(x,y)$，其中

$$ x = [sim(I_1,S_1),…,sim(I_m,S_m); sim(I_1,S_{j1}),…,sim(I_m,S_{jm})]
$$

$j1,…jm$ 为打乱的 $1,…,m$

$$ y = [1,…,1;-1,…,-1]
$$

实验（Experiments）

数据集

本文使用了 3 个数据集：MS COCO, SBU 和 Pinterest5M

其中 MS COCO 的描述比较正式，而 SUB 和 Pin 的描述来源于非正式的帖子；MS COCO 和 Pin 对每个图片有多个描述，本文只保留其中一个，以防这些文本在匹配时只使用图像信息。

MS COCO

80 个类别，每幅图像有 5 个高质量的描述。本文使用的是 MS COCO 2014，train/validation/test 的划分参考[23][28]，分别包含 586k/10k/20k examples。

SBU

1M image-caption pairs collected from Flickr, with only one caption per image. 本文随机将数据划分成 900k/50k/50k 作为 train/validation/test

Pinterest5M

该数据集来自[15]，实际只有 3.9M 的图片可下载，本文随机将数据划分成 3.8M/50k/50k 作为 train/validation/test

所有数据都做了 lower 处理，并使用 Stanford Tokenizer [14] 序列化。

超参数选择（Hyperparameter selection）

embeddings_size = 128
epoch = 10(COCO, SBU) or 5(Pinterest5M)

评价（Evaluation）

本文的目的是构建一个良好的 text embedding，能够融合相关的图像信息。为了评价知识从图像到文本的转移效果，本文使用了一组来自2014和2015年 SemEval 比赛的文本语义相似数据集 [18]。

此外，还使用了两个定制数据集：COCO-Test 和 Pin-Test。它们分别来自 COCO 和 Pinterest5M，从中随机抽取 1000 条语义相关的标题(来自同一图像)和 1000 条来自不同图像的非相关标题。

Ablation studies

这一节主要讲的是不同因素对模型的影响，相当于控制变量。

主要讲了以下几个因素：

  1. 文本部分的 Encoder

  2. Loss 函数

  3. 数据集

  4. 句向量与词向量

    以上的实验都是基于 sentence-level 的 embedding 产生的，而以往的研究都是试图向 word-level 的 embedding 中注入图像信息。这一节主要的目的就是比较这两种方案的优劣。

    本文分析了 sentence embedding 的优势：单独训练 word embedding 会迫使每个词都接近相应的图像，而 sentence embedding 则提供了使 word embedding 互补的机会，即每个词都解释了图像的一部分，并捕捉到了共现（co-occurences）。
